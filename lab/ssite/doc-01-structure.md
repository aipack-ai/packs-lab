# Project Directory Structure

This document outlines the standard directory layout and the flow of data from source to distribution in an `aipack-site` project.

## Root Layout

The project follows a modular structure where source code is separated from static content and final distribution.

- `src/`: TypeScript source code for the application logic and Web Components.
- `css/`: CSS source files (typically using modern CSS nesting).
- `content/`: The "Source of Truth" for `ssite`. Contains static assets and serves as the destination for build artifacts.
- `_site/`: The final distribution directory generated by `ssite`, ready for deployment.
- `scripts/`: Node.js build orchestration scripts.
- `lab/`: (Optional) Documentation and research files.

## The content/ Directory

The `content/` directory plays a dual role in this architecture:

1. **Static Source**: It contains all static files that do not require transpilation, such as `.html` templates, `.md` files, images, and fonts.
2. **Build Target**: The JS/TS and CSS build tools output their final bundles into specific subdirectories within `content/`.

### Build Artifacts in content/

- `content/js/all-bundle.js`: Bundled and transpiled JavaScript from `src/`.
- `content/css/all-bundle.css`: Bundled and processed CSS from `css/`.

## Data Flow

1. **Develop**: Developers work in `src/` and `css/`.
2. **Build**: Tools (`rolldown`, `lightningcss`) compile source files into `content/js/` and `content/css/`.
3. **Process**: `ssite` processes the `content/` directory, handling any dynamic content (like markdown to HTML) and moving everything into `_site/`.
4. **Deploy**: The contents of `_site/` are deployed to the host (e.g., S3).

## Git Management

To keep the repository clean and avoid merge conflicts on generated files, the following patterns should be added to `.gitignore`:

- `node_modules/`
- `_site/`
- `content/js/all-bundle.js*`
- `content/css/all-bundle.css*`

Note: Committing the generated bundles is generally discouraged unless specifically required by a specialized deployment workflow.
